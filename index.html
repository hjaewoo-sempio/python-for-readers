AI 기반 코딩 심화: 파이썬 코드 읽기 매뉴얼

코드 리터러시(Code Literacy) 매뉴얼 v2.1

AI가 작성한 코드를 "읽고, 검증하고, 수정"하기 위한 가이드북입니다.
각 챕터별 핵심 이론과 5가지 실전 예시를 포함하고 있습니다.

Chapter 1

Import & 라이브러리 (도구 상자)

📖 이론: 도구를 책상 위로 꺼내기

파이썬은 "모든 기능"을 한 번에 메모리에 올리지 않습니다. 필요한 기능이 담긴 "도구 상자(라이브러리)"를 명시적으로 가져와야(Import) 사용할 수 있습니다.

AI가 작성한 코드가 시작하자마자 에러(NameError)가 난다면, 99%는 이 Import 구문이 빠져서입니다. "Pandas를 써줘"라고 했지만 import pandas를 빼먹는 경우가 종종 있습니다.

💡 실전 예시 5선

1. 별명(Alias) 붙이기 (가장 중요)
'pandas'라고 매번 치기 귀찮아서 'pd'라는 별명을 붙입니다. 이것은 전 세계 개발자들의 약속(Convention)입니다. AI도 항상 이렇게 씁니다. 
import pandas as pd  # pandas를 가져오되, 이름표를 'pd'로 붙임
import numpy as np   # numpy를 가져오되, 이름표를 'np'로 붙임

# 사용: pandas.read_csv가 아니라 pd.read_csv로 씁니다.
data = pd.read_csv("data.csv")
2. 도구 상자에서 '특정 도구'만 꺼내기
random이라는 큰 상자를 다 가져오지 않고, 그 안에 있는 randint(랜덤 정수 뽑기) 기능 하나만 쏙 꺼냅니다. 코드가 깔끔해집니다. 
# random 상자에서 randint 기능만 꺼내옵니다.
from random import randint

# 이제 random.randint()라고 안 하고 바로 씁니다.
lucky_num = randint(1, 100)
3. 기본 Import (운영체제 기능)
별명 없이 그냥 가져오는 경우입니다. 파일 경로를 다루거나 폴더를 만들 때 쓰는 os 라이브러리가 대표적입니다. 
import os

# 현재 폴더에 있는 파일 목록을 가져옵니다.
files = os.listdir(".")
4. JSON 데이터 처리
웹사이트나 API와 통신할 때 데이터는 대부분 JSON 형식입니다. 이를 파이썬 딕셔너리로 바꾸기 위해 필수적입니다. 
import json

raw_data = '{"name": "Kim", "age": 30}'  # 텍스트 형태
data_dict = json.loads(raw_data)          # 파이썬 딕셔너리로 변환
5. 시간 관련 기능
프로그램을 잠시 멈추거나(딜레이), 실행 시간을 잴 때 사용합니다. 크롤링할 때 서버 과부하를 막기 위해 자주 씁니다. 
import time

print("데이터 요청 중...")
time.sleep(3)  # 3초 동안 아무것도 안 하고 쉽니다.
print("완료!")
Chapter 2

변수 (이름표 붙이기)

📖 이론: 타입(Type)을 조심하세요

파이썬은 변수를 만들 때 타입을 지정하지 않아도 됩니다(int a = 5 처럼 안 써도 됨). 하지만 "문자"와 "숫자"는 엄격히 구분합니다. 엑셀에서 읽어온 데이터가 눈에는 숫자로 보여도, 실제로는 문자(String)인 경우가 많습니다. 이때 더하기를 하면 계산이 아니라 '글자 이어붙이기'가 됩니다.

💡 실전 예시 5선

1. 문자와 숫자의 차이 (가장 흔한 실수)
따옴표("")로 감싸져 있으면 무조건 문자입니다. 
a = "100"  # 문자 100
b = 100    # 숫자 100

# print(a + b)  # 에러 발생! 문자와 숫자는 더할 수 없음
print(a + a)    # 결과: "100100" (글자 이어붙이기)
print(b + b)    # 결과: 200 (산술 연산)
2. f-string (AI가 사랑하는 문자열 조립법)
변수를 문장 사이에 끼워 넣을 때 가장 세련된 방법입니다. 문자열 앞에 f를 붙이고 중괄호 {} 안에 변수를 넣습니다. 
name = "박대리"
score = 95

# 옛날 방식: "이름: " + name + ", 점수: " + str(score) (복잡함)
# 요즘 방식 (f-string):
msg = f"이름: {name}, 점수: {score}점"
3. 타입 변환 (Casting)
문자를 숫자로(int), 혹은 숫자를 문자로(str) 바꿔야 할 때가 있습니다. AI 코드에서 자주 보게 됩니다. 
input_val = "500"
# 계산을 위해 숫자로 변경
real_num = int(input_val) 
result = real_num + 100  # 600
4. 불리언 (참/거짓)
True와 False는 앞글자가 대문자여야 합니다. 주로 스위치(ON/OFF) 역할로 씁니다. 
is_active = True
has_error = False

# AI는 주로 이렇게 씁니다
if is_active:
    print("시스템 가동")
5. 변수명 규칙 (Snake Case)
파이썬에서는 변수 이름을 지을 때 띄어쓰기 대신 언더바(_)를 쓰는 것이 관례입니다. 
user_email_address = "a@b.com" # (O) 파이썬 스타일
userEmailAddress = "a@b.com"   # (X) 자바/자바스크립트 스타일
# 문법 에러는 아니지만, 파이썬 개발자들은 전자를 선호합니다.
Chapter 3

들여쓰기 & 범위 (소속 관계)

📖 이론: 공백이 문법이다

다른 언어는 중괄호 {}로 코드 뭉치를 묶지만, 파이썬은 **들여쓰기(Indentation)**로 묶습니다. "누구 밑에 줄이 맞춰져 있는지"가 실행 여부를 결정합니다. 보통 스페이스바 4칸 혹은 탭(Tab) 1번을 씁니다. AI 코드를 복사해 올 때 줄이 틀어지면 IndentationError가 발생하니 주의하세요.

💡 실전 예시 5선

1. If문의 소속
조건이 참일 때만 실행되어야 하는 코드는 반드시 들여쓰기를 해야 합니다. 
if score > 80:
    print("합격")        # if문 안쪽 (조건 만족 시 실행)
    print("축하합니다")  # if문 안쪽
print("종료")            # if문 바깥 (무조건 실행)
2. 반복문의 소속 (가장 중요)
들여쓰기 된 부분은 N번 반복되고, 들여쓰기가 끝난 부분은 반복이 끝난 후 딱 1번 실행됩니다. 이 차이를 아는 것이 핵심입니다. 
for i in [1, 2, 3]:
    print("파일 처리 중...") # 3번 실행됨

print("완료 알림 메일 전송")  # 1번 실행됨 (반복문 밖)
3. 함수 정의의 소속
함수의 내용물은 반드시 함수 이름 아래에 들여쓰기 되어야 합니다. 
def my_func():
    a = 10
    b = 20
    return a + b  # 여기까지 함수에 포함됨

c = 30            # 함수 바깥임
4. 중첩 구조 (Double Indentation)
If문 안에 For문이 있거나, For문 안에 If문이 있는 경우 들여쓰기가 두 번 들어갑니다. 
if is_working_day:
    print("출근")
    for hour in [9, 10, 11]:
        print(f"{hour}시 업무") # 두 번 들여쓰기 됨
5. 흔한 실수 (Indentation Error)
아래 코드는 에러가 납니다. 함수 내용물이 없거나 줄이 안 맞기 때문입니다. 
def hello():
print("안녕") # 에러! 함수 내용물인데 들여쓰기가 안 됨

if True:
    print("참")
  print("거짓") # 에러! 윗줄과 들여쓰기 칸 수가 다름
Chapter 4

리스트 & 딕셔너리 (데이터 정리함)

📖 이론: 순서냐 이름이냐

데이터를 여러 개 담을 때 쓰는 두 가지 그릇입니다.

  • 리스트(List) []: 순서가 중요합니다. "0번째", "1번째" 처럼 번호(Index)로 찾습니다.
  • 딕셔너리(Dict) {}: 이름표(Key)가 중요합니다. "이름이 뭐야?", "나이가 몇이야?" 처럼 키워드로 찾습니다.

💡 실전 예시 5선

1. 리스트 만들기 및 조회
컴퓨터는 숫자를 0부터 셉니다. 뒤에서부터 셀 때는 -1을 씁니다. 
team = ["A", "B", "C"]

first = team[0]   # "A"
last = team[-1]   # "C" (뒤에서 첫 번째)
2. 리스트에 내용 추가하기 (append)
빈 리스트를 만들고 append로 하나씩 채워 넣는 패턴은 매우 흔합니다. 
results = []          # 빈 리스트
results.append(10)    # [10]
results.append(20)    # [10, 20]
3. 딕셔너리 만들기 및 조회
'키(Key)'와 '값(Value)'의 쌍으로 이루어집니다. 엑셀의 한 행과 비슷합니다. 
user = {
    "name": "홍길동",
    "age": 30
}
print(user["name"])  # 결과: 홍길동
4. 딕셔너리 값 수정/추가
이미 있는 키를 부르면 수정이 되고, 없는 키를 부르면 새로 추가됩니다. 
info = {"price": 100}

info["price"] = 200   # 수정
info["sale"] = True   # 추가 (sale이라는 키가 새로 생김)
5. 리스트 안에 딕셔너리 (List of Dicts)
실무 데이터의 90%가 이 형태입니다. 엑셀 표를 그대로 코드로 옮기면 이렇게 됩니다. 
data = [
    {"date": "1/1", "val": 100},  # 0번 행
    {"date": "1/2", "val": 150}   # 1번 행
]

# 1번 행의 'val' 값 가져오기
target = data[1]["val"]  # 150
Chapter 5

조건문 (판단 능력)

📖 이론: 질문하고 선택하기

AI가 똑똑해 보이는 이유는 조건문을 잘 쓰기 때문입니다. 특히 파이썬의 in 연산자는 매우 직관적입니다. "리스트 안에 있니?", "문장 안에 포함되니?"를 영어 문장처럼 물어볼 수 있습니다.

💡 실전 예시 5선

1. 숫자 크기 비교
가장 기본적인 조건문입니다. if, elif(그렇지 않고 만약에), else(나머지 전부)를 씁니다. 
stock = 5

if stock > 10:
    print("충분")
elif stock > 0:
    print("부족 (주문 필요)")
else:
    print("품절")
2. 포함 여부 확인 (문자열)
in을 사용하면 문장 안에 특정 단어가 있는지 쉽게 찾을 수 있습니다. 
email = "my.name@gmail.com"

if "@" in email:
    print("이메일 형식입니다.")
3. 포함 여부 확인 (리스트)
리스트 안에 특정 항목이 있는지 확인할 때도 in을 씁니다. 반대는 not in입니다. 
vip_users = ["user1", "user2"]
current = "user3"

if current in vip_users:
    print("VIP 입장")
else:
    print("일반 입장")
4. 논리 연산자 (and / or)
두 가지 이상의 조건을 동시에 검사할 때 사용합니다. 
age = 25
has_card = True

# 나이가 20 이상 '이고' 카드가 있어야 함
if age >= 20 and has_card:
    print("결제 가능")
5. 데이터 존재 여부 체크 (Truthy/Falsy)
파이썬에서 비어있는 리스트는 거짓(False)으로 취급됩니다. 이를 이용해 데이터가 있는지 없는지 아주 짧게 검사합니다. 
data = []  # 빈 리스트

if not data:  # "데이터가 없으면" 이라는 뜻
    print("처리할 데이터가 없습니다.")
Chapter 6

반복문과 Pythonic Style

📖 이론: 한 줄의 미학

반복문은 코드를 획기적으로 줄여줍니다. 특히 파이썬에는 **"리스트 컴프리헨션(List Comprehension)"**이라는 독특한 문법이 있습니다.

[x*2 for x in nums] 처럼 대괄호 안에 반복문이 들어있는 형태입니다. AI가 코드를 짜주면 80% 확률로 이 문법을 사용합니다. "복잡해 보이지만, 그냥 반복해서 리스트를 새로 만드는 거구나" 하고 이해하시면 됩니다.

💡 실전 예시 5선

1. 기본 For Loop
리스트 안에 있는 것을 하나씩 꺼내서 `item`이라는 변수에 담고 반복합니다. 
files = ["a.txt", "b.txt"]
for item in files:
    print(f"파일 삭제: {item}")
2. 횟수 지정 반복 (range)
리스트 없이 그냥 "10번 반복해"라고 하고 싶을 때는 range(숫자)를 씁니다. 
# 0, 1, 2, 3, 4 (총 5회)
for i in range(5):
    print(f"{i}번째 시도")
3. 딕셔너리 반복 (.items)
키와 값을 동시에 꺼내면서 반복하고 싶을 때 씁니다. 
scores = {"Math": 90, "Eng": 80}

for subject, score in scores.items():
    print(f"과목: {subject}, 점수: {score}")
4. 리스트 컴프리헨션 (Pythonic!)
"기존 리스트를 가공해서 새 리스트 만들기"를 한 줄로 줄인 것입니다. [결과 for 변수 in 리스트] 형태로 읽습니다. 
nums = [1, 2, 3]

# "nums를 돌면서 각 숫자(n)에 10을 곱해라"
result = [n * 10 for n in nums]
# 결과: [10, 20, 30]
5. 조건부 리스트 컴프리헨션 (Pythonic!)
반복을 하면서 필터링(조건)까지 동시에 하는 경우입니다. AI가 데이터 전처리할 때 가장 많이 쓰는 패턴입니다. 
files = ["img.png", "data.csv", "logo.png"]

# "파일 목록을 돌면서, 'png'가 포함된 파일만 골라라"
png_files = [f for f in files if "png" in f]
# 결과: ["img.png", "logo.png"]
Chapter 7

함수 (Input과 Output)

📖 이론: 자판기 설치 vs 사용

함수는 자판기와 같습니다. 돈을 넣으면(Input) 음료가 나옵니다(Output). 중요한 건 설계(Def)와 사용(Call)을 구분하는 것입니다.

AI가 코드를 짤 때 def ... 부분은 "이런 기능을 만들 거야"라고 설계도를 그리는 부분입니다. 실제 실행은 코드 맨 아래쪽이나 def 바깥에서 일어납니다.

💡 실전 예시 5선

1. 기본 함수 (Input O, Output O)
가장 표준적인 함수입니다. 값을 받아서 가공 후 돌려줍니다(Return). 
# [설계]
def double(x):
    return x * 2

# [사용]
res = double(10)
print(res) # 20
2. 입력값이 여러 개인 함수
콤마(,)로 구분해서 여러 재료를 넣을 수 있습니다. 순서를 맞춰야 합니다. 
def add(a, b):
    return a + b

result = add(3, 5)
3. 기본값(Default) 설정
입력값을 안 넣었을 때 자동으로 적용될 값을 정해둘 수 있습니다. 
def greet(name, msg="안녕하세요"):
    print(f"{name}님, {msg}")

greet("철수")              # "철수님, 안녕하세요"
greet("영희", "반가워요")  # "영희님, 반가워요"
4. 반환값(Return)이 없는 함수
단순히 화면에 출력하거나 파일을 저장하고 끝나는 함수는 return이 없습니다. 이때 결과를 변수에 담으면 None(없음)이 됩니다. 
def log(msg):
    print(f"[LOG] {msg}")
    # return 없음

a = log("시작")
# print(a) -> 결과: None
5. 흔한 실수 (Return vs Print)
Print는 화면에 보여줄 뿐, 값을 전달하지 않습니다. 계산을 이어가려면 반드시 Return해야 합니다. 
def bad_add(a, b):
    print(a + b) # 화면엔 5가 보이지만...

res = bad_add(2, 3)
# final = res + 10  -> 에러! res는 비어있음(None)
Chapter 8

클래스 (사용법 위주)

📖 이론: 점(.)의 의미

AI 관리자가 클래스를 직접 설계할 일은 거의 없습니다. 하지만 변수.함수() 형태는 매일 보게 됩니다.

여기서 점(.)은 "네가 가지고 있는 기능을 써라"라는 명령어입니다. "문자열아(.upper), 대문자로 변해라", "리스트야(.append), 데이터를 추가해라" 처럼, 데이터 타입마다 가지고 있는 전용 기능(메서드)들이 있습니다.

💡 실전 예시 5선

1. 문자열(String)의 기능
문자열은 대소문자 변환, 쪼개기(split), 공백제거(strip) 등의 기능을 내장하고 있습니다. 
text = " apple "

clean = text.strip()  # "apple" (공백제거)
big = clean.upper()   # "APPLE" (대문자)
2. 리스트(List)의 기능
리스트는 순서 정렬(sort), 뒤집기(reverse), 추가(append) 등의 기능을 가집니다. 
nums = [3, 1, 2]
nums.sort()  # 리스트 자신이 정렬됨
# 결과: [1, 2, 3]
3. 딕셔너리(Dict)의 기능
키만 뽑기(keys), 값만 뽑기(values) 등의 기능이 있습니다. 
data = {"name": "A", "age": 20}

keys = list(data.keys()) 
# 결과: ["name", "age"]
4. 사용자 정의 클래스 읽기 (__init__)
AI가 class를 정의했다면 __init__ 부분을 보세요. 이 클래스가 처음 생성될 때 어떤 재료(변수)를 필요로 하는지 알 수 있습니다. 
class User:
    # 생성자: 처음 만들 때 이름(name)을 내놓으라고 함
    def __init__(self, name):
        self.name = name

u = User("철수") # 그래서 만들 때 "철수"를 넣어줌
5. 객체 속성 접근
객체 안에 저장된 변수는 객체.변수명으로 꺼낼 수 있습니다. 
# 위 예시에서 이어짐
print(u.name)  
# 결과: "철수"
Chapter 9

실전 라이브러리 (Pandas & Requests)

📖 이론: 사무 자동화의 양대 산맥

AI를 이용한 업무 자동화는 결국 이 두 가지로 귀결됩니다.

  • Pandas: 엑셀 파일 읽기, 데이터 필터링, 통계 내기 (엑셀 대체)
  • Requests: 웹사이트 접속, API 데이터 가져오기 (인터넷 연결)

💡 실전 예시 5선

1. Pandas: 엑셀 읽고 확인하기
read_excel로 불러오고, head()로 상위 5줄만 눈으로 확인합니다. 데이터 검증의 첫 단계입니다. 
import pandas as pd

df = pd.read_excel("sales.xlsx")
print(df.head())  # 데이터가 잘 들어왔나 눈으로 확인
2. Pandas: 데이터 필터링
엑셀의 '필터' 기능과 같습니다. 조건을 만족하는 행만 뽑아서 새 변수에 저장합니다. 
# '금액' 컬럼이 10000 이상인 행만 추출
high_value = df[df['금액'] >= 10000]
3. Pandas: 새 컬럼(열) 만들기
엑셀의 수식(C열 = A열 * B열)과 같습니다. 간단하게 곱하기만 하면 모든 행에 적용됩니다. 
# '총액'이라는 새 열을 만듭니다.
df['총액'] = df['단가'] * df['수량']
4. Requests: 웹 연결 확인
status_code가 200이면 성공, 404면 페이지 없음, 500이면 서버 오류입니다. 
import requests

res = requests.get("https://google.com")
if res.status_code == 200:
    print("접속 성공")
5. Requests: JSON 데이터 가져오기
API에서 가져온 데이터를 .json()으로 변환하면 파이썬의 딕셔너리처럼 쓸 수 있습니다. 
url = "https://api.test.com/users"
res = requests.get(url)

data = res.json()  # 딕셔너리(혹은 리스트)로 변환
print(data["name"])
